# Git4Beginner - 交互式Git学习平台SPA版本：设计说明文档

## 1. 项目概述

### 1.1. 项目名称
**Git4Beginner——零基础Git交互式学习平台**

### 1.2. 项目目标

本项目旨在为零基础的计算机初学者提供一个友好、交互式的Git学习平台。通过结合理论教程、模拟实验和在线测验，帮助用户安全、高效地掌握 Git 的核心概念与操作。

### 1.3. 目标用户

* 计算机科学专业的学生
* 对版本控制完全没有概念的编程初学者
* 希望在动手实践中学习Git的用户

## 2. 视觉设计 (VI) & 风格指南

### 2.1. 整体风格
* **现代、简洁、专业、富有教育感**。
* 界面大量使用**圆角**（`rounded-lg`, `rounded-xl`）、**阴影**（`shadow-lg`）来营造卡片式布局，增加层次感。

### 2.2. 字体 (Typography)
* **UI 字体**: `Inter` (sans-serif)。用于所有非代码文本，确保易读性。
* **代码字体**: `JetBrains Mono` (monospace)。用于所有代码片段、教程中的 `<code>`, `<pre>` 标签以及模拟终端，提供清晰的编程字体体验。

### 2.3. 色板 (Color Palette)

| 颜色名称 | 色值 (Hex) | 用途说明 |
| :--- | :--- | :--- |
| **主色 (Main)** | `#1e3a8a` | 深蓝色。用于品牌标识、主要按钮、高亮标题。代表专业、稳定。 |
| **辅助色/成功 (Auxiliary/Success)** | `#059669` | 绿色。用于“立即开始”按钮、进度条、成就提示、测验正确答案。代表进度、成功。 |
| **强调色 (Accent)** | `#ea580c` | 橙色。用于功能特性模块的标题，吸引注意力。 |
| **代码区背景 (Code BG)** | `#0f172a` | 深黑蓝色。用于模拟终端和代码块背景，提供高对比度的阅读环境。 |
| **代码区文字 (Code Text)** | `#f1f5f9` | 亮灰色。 |
| **页面背景 (Page BG)** | `#f1f5f9` | 浅灰色。 |
| **内容背景 (Content BG)** | `#ffffff` | 白色。用于卡片和主要内容区域背景。 |
| **基础文本 (Base Text)** | `#0f172a` | 深石板蓝。 |

## 3. 页面结构与功能

本应用是一个**单页面应用（SPA）**，通过JavaScript控制四个核心页面的显示与隐藏。

### 3.1. 导航栏 (Nav)
* **样式**: `sticky top-0 z-50`，始终固定在页面顶部，背景白色，带有阴影。
* **功能**: 包含 "首页", "Git教程", "开始实验", "在线测验" 四个链接。点击后调用 `showPage(id)` 函数切换页面视图。

### 3.2. 页面 1: 首页 (`#page-home`)
* **布局**: 多区块垂直布局。
* **核心组件**:
    * **Hero 区**: 使用主色背景，`anime.js` 驱动标题和副标题的淡入动画。
    * **功能特性**: 采用三栏卡片布局，卡片具有 `hover:scale-105` 的交互效果。
    * **学习路径 & 雷达图**: 采用两栏布局。
        * *学习路径* (`#learning-path-list`): 垂直列表，通过 JS 动态更新样式 (`step-active`, `step-completed`) 来实时反映用户在实验页的进度。
        * *技能雷达图* (`#skills-radar-chart`): 使用 **ECharts** 库绘制，数据源 (`skillData`) 会根据实验完成情况自动更新。

### 3.3. 页面 2: 教程页 (`#page-tutorial`)
* **布局**: 单栏内容页，最大宽度 `max-w-4xl`，居中显示。
* **核心组件**:
    * **静态内容**: 包含标准排版元素（`h2`, `h3`, `p`, `ul`, `ol`, `pre`, `code` 等），样式已在 `<style>` 中全局定义。
    * **命令速查表**:
        * *交互*: 表格的每一行 (`tr.command-lookup`) 都绑定了点击事件。
        * *功能*: 点击后触发 `showCommandModal(commandKey)` 函数，弹出一个模态窗口。
    * **命令详情弹窗** (`#command-modal-overlay`):
        * *样式*: 黑色半透明遮罩层，内容卡片居中显示。
        * *功能*: 动态填充所点击命令的详细信息（语法、详情、示例）。

### 3.4. 页面 3: 实验页 (`#page-lab`)
* **布局**: 三栏式布局（移动端适配为垂直布局），高度占满视口 (`height: calc(100vh - 64px)`).
* **核心组件**:
    * **左侧栏 (步骤)**:
        * *进度条* (`#lab-progress-bar`): 由 `anime.js` 驱动，根据步骤完成度平滑更新宽度。
        * *步骤列表* (`#lab-steps-container`): 动态生成。当前步骤高亮显示 (`step-active`)，已完成步骤添加标记 (`step-completed`).
    * **中间 (模拟终端)**:
        * *输出区* (`#terminal-output`): 可滚动区域，用于显示系统提示和命令回显。
        * *输入区* (`#terminal-input`): 核心交互区。监听 `keydown` 事件，回车时获取命令。
        * *校验逻辑*: 使用正则表达式 `new RegExp(step.match).test(cmd)` 校验用户输入，而非简单的字符串匹配，提供了更高的灵活性。
    * **右侧栏 (提示)**: 显示当前步骤对应的提示信息 (`#lab-hint-container`).
    * **成就提示** (`#achievement-toast`):
        * *样式*: `fixed` 定位在右下角，使用辅助色（绿色）背景。
        * *交互*: 默认隐藏。完成实验时通过 `showAchievement()` 函数触发，`anime.js` 驱动其滑入和滑出动画。

### 3.5. 页面 4: 测验页 (`#page-quiz`)
* **布局**: 单栏卡片式，居中显示。
* **核心组件**:
    * **问题区**: 动态加载当前问题 (`#question-text`) 和选项 (`#quiz-options`).
    * **选项** (`.quiz-option`): 按钮形态。点击后添加 `.selected` 样式。
    * **反馈机制**:
        * 提交后，所有选项按钮禁用并显示 `.disabled` 样式。
        * 正确答案添加 `.correct` (绿色) 样式，错误选项添加 `.incorrect` (红色) 样式。
        * `#quiz-feedback` 区域显示具体的文字反馈。
    * **结果页** (`#quiz-results`): 测验全部完成后显示，展示用户的最终得分。

## 4. 所用技术栈

* **核心**: HTML5, CSS3, ES6+ (Vanilla JavaScript)。
* **CSS 框架**: Tailwind CSS (通过 CDN 引入)。
* **JavaScript 库**:
    * **anime.js**: 用于实现平滑的 UI 动画效果（首页 Hero、进度条、成就提示）。
    * **ECharts**: 用于绘制首页的技能雷达图。
